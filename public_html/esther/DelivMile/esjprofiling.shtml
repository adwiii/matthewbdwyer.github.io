CSCE 425/825 - Compiler Construction

Milestone Grad : Profiling for ESJ

Your task in this milestone is to extend the existing code generation support for ESJ developed in Milestone 5 to achieve rudimentary performance profiling support. Tools like jProfiler provide sophisticated mechanisms for recording information about a program's execution behavior and presenting that information to developers so that they can detect, for example, performance hot-spots and memory leaks.

This milestone consists of two parts.

First, you will implement a much more rudimentary capability much like that in gprof. More specifically, you will modify your code generator to insert bytecodes that record the number of times that each block of code in the code is executed. A block of code is a sequential run of statements. Control structures like, if and for, introduce branches that interrupt sequential runs of statements. Your solution must print a list of blocks that is ordered based on execution count - from high to low.

Second, you will optimize your code generation strategy to minimize the number of inserted bytecodes. This optimization will be driven by an analysis of statement dominator information; see the Dragon book Chapter 9.6.1 for a definition. By calculating dominators, you will be able to determine whether some of the inserted bytecodes can be removed without losing any block execution count information.

Finally, you will develope a set of test programs that illustrate the correctness and effectiveness of your solution. For each test program, you will provide the number of bytecodes generated and the ordered list of blocks; the latter information should be identical for both unoptimized and optimized code generation strategies.
